## 🎯 درخواست کاربر

> "الان من متوجه شدم که مشکل از کجاست. به عنوان مثال توی سوال 'درآمد استانی اختصاصی وزارت آموزش و پرورش از سال 1398 تا 1403' ما برای رسیدن به جواب داریم اول توی عنوان_دستگاه_اصلی میگردیم که درست هم هست. جوابی که میدیم هم بر همین اساس هستش. ولی توی جدول و detail_rows فقط داریم دیتاهایی رو میاریم که عنوان_دستگاه_اجرایی شون هم همون 'وزارت آموزش و پرورش' باشه ولی باید توی این 2 تا پارامتر هم بریم چیزهایی رو نشون بدیم که عنوان_دستگاه_اصلی شون 'وزارت آموزش و پرورش' باشه."

### انتظار:
- ✅ `detail_rows` شامل همه rows با `عنوان_دستگاه_اصلی = "وزارت آموزش و پرورش"` باشد
- ✅ شامل rows که `عنوان_دستگاه_اجرایی ≠ "وزارت آموزش و پرورش"` هم باشد (مثلاً ادارات استانی)

### 1. بررسی SQL Query

**SQL اصلی (Summary)**:
```sql
SELECT "سال", SUM(...) as total_amount 
FROM manabe3_sheet1 
WHERE (
  TRANSLATE("عنوان_دستگاه_اجرایی", ...) ILIKE '%وزارت آموزش و پرورش%'
  OR 
  TRANSLATE("عنوان_دستگاه_اصلی", ...) ILIKE '%وزارت آموزش و پرورش%'
)
AND TRANSLATE("سال", ...) IN ('1398', '1399', '1400', '1401', '1402', '1403')
GROUP BY "سال"
```

**SQL جزئیات (Detail)**:
```sql
SELECT "عنوان_دستگاه_اصلی", "عنوان_دستگاه_اجرایی", "کد_دستگاه_اجرایی", "سال", ...
FROM manabe3_sheet1 
WHERE (
  TRANSLATE("عنوان_دستگاه_اجرایی", ...) ILIKE '%وزارت آموزش و پرورش%'
  OR 
  TRANSLATE("عنوان_دستگاه_اصلی", ...) ILIKE '%وزارت آموزش و پرورش%'
)
AND TRANSLATE("سال", ...) IN ('1398', '1399', '1400', '1401', '1402', '1403')
LIMIT 500
```

✅ **WHERE Clause صحیح است** - هر دو ستون `اصلی` و `اجرایی` با OR چک می‌شوند!

### 2. بررسی Filtering

**کد قبلی (اشتباه)**:
```python
# اگر هر دو parent match و exact entity match داریم،
# فقط ردیف‌هایی که exact entity match دارند را برگردان
if matching_parents and exact_entity_matches:
    filtered_matches = [row for row in detail_rows 
                        if normalize(row['عنوان_دستگاه_اجرایی']) in exact_entity_matches]
    return filtered_matches  # ❌ فقط exact entity!
```

**کد جدید (صحیح)**:
```python
# اگر parent match داریم، همه rows با آن parent را نگه دار
if matching_parents:
    filtered_matches = [row for row in detail_rows 
                        if normalize(row['عنوان_دستگاه_اصلی']) in matching_parents]
    return filtered_matches  # ✅ همه rows با parent matching!
```

✅ **Fix شد** - حالا همه rows با `عنوان_دستگاه_اصلی` matching نگه داشته می‌شوند!

آیا ستون‌های دیگری مثل `استان` یا `نام_استان` یا `عنوان_قسمت` وجود دارد که تفکیک را نشان دهند؟

```sql

  "عنوان_دستگاه_اصلی",
  "عنوان_دستگاه_اجرایی", 
  "عنوان_قسمت",
  "عنوان_بخش",
```
