## 🔧 مشکلات شناسایی شده

### 1. مشکل Routing به Database
- بسیاری از query‌ها به جای استفاده از database، از RAG استفاده می‌کنند
- Query‌های مربوط به درآمد (`incomes_sheet1`) به درستی route نمی‌شوند

### 2. مشکل Entity Matching
- سیستم نمی‌تواند entity‌های پیچیده را در database پیدا کند
- مثال: "سازمان سنجش بند ج" با عنوان کامل در database match نمی‌شود

### 3. مشکل Default Year
- وقتی سال ذکر نشده، سیستم سال 1403 را به صورت پیش‌فرض اضافه نمی‌کند
- مثال: "بودجه دانشگاه تهران" بدون سال

### 4. مشکل SQL Query در Response
- SQL query در پاسخ API موجود نیست
- این باعث می‌شود نتوانیم ببینیم چه query‌ای اجرا شده

### 5. مشکل زبان پاسخ
- برخی پاسخ‌ها به انگلیسی هستند
- باید همه پاسخ‌ها فارسی باشند

### 6. مشکل "No results found"
- بسیاری از query‌ها خطای HTTP 500 با پیام "No results found" می‌دهند
- این می‌تواند به دلیل:
  - Entity matching ناموفق
  - SQL query خطا
  - عدم وجود داده در database

## 💡 پیشنهادات بهبود

### 1. بهبود Entity Matching
- استفاده از fuzzy matching برای entity‌های پیچیده
- Normalization بهتر نام entity‌ها
- استفاده از aliases برای entity‌های با نام‌های مختلف

### 2. بهبود Routing
- اطمینان از اینکه query‌های درآمد به `incomes_sheet1` route می‌شوند
- بررسی logic تشخیص جدول در `detect_target_table`

### 3. بهبود Default Year
- اطمینان از اینکه وقتی سال ذکر نشده، سال 1403 اضافه می‌شود
- بررسی `_add_default_year_if_missing` در `query_orchestrator.py`

### 4. اضافه کردن SQL به Response
- SQL query را در response API اضافه کنیم
- این برای debugging و analysis مفید است

### 5. بهبود زبان پاسخ
- اطمینان از اینکه همه پاسخ‌ها فارسی هستند
- بررسی system prompt برای `budget_financial`

### 6. بهبود Error Handling
- به جای HTTP 500، پیام‌های خطای واضح‌تر برگردانیم
- Logging بهتر برای debugging
